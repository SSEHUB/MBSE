Elevator simulator help document


The program window shows the simulation of the previously configured elevators/floors.

Below each elevator there is a panel shown information on each elevator including the control buttons within the elevator.

Default floor numbers or floor names read in from a file are shown left to the floors.


            Indicate elevator request:

Call an elevator with the outer buttons (if configured) as well as the lower right buttons showing the panel inside the elevator.

            Cancel elevator request:

By pressing the call button used befor a second time, you can cancel your actual elevator request.

            Emergency/fire mode (behavior of built-in controllers):

By pressing the emergency/fire mode the calling floor receives highest priority and the elevator shall ignore all unprocessed elevator requests.
If the fire mode is activated from inside the elevator, then the buttons inside the elevator become highest priority and outside buttons are ignored.

            Floor locking (behavior of built-in controllers):

Using a right mouse-button on the request button you can lock an entire floor, which can then not used for elevator calling. A floor can be unlocked by another right mouse click on the same request button.

            Privileged floors (behavior of built-in controllers, may be incomplete):

The floor names file may assign authorization levels to individual floors. If you want an elevator to move to a privileged floor, you must authorize the access using the key button . Please enter your keycode in the input window to unlock the floor.

            Door control (if configured, behavior of default controllers):

By pressing the button with you may explicitly open or close the elevator doors. As usual, these buttons are inside the elevator (if configured) and allow you to bypass the configured door delays be closing the doors earlier or by keeping the doors open longer.
